# Roana

> **Walk your world. 走你自己的路。** 一个为盲人和低视力用户设计的开源辅助导航系统。

> 🌐 English version: [README.md](README.md) · 当前页面:简体中文

Roana 把一部智能手机(以及未来的智能眼镜)变成一个实时环境感知伙伴:端侧计算机视觉识别障碍物和可行走区域,然后通过**骨传导语音**(告诉你"周围有什么")和**腕部震动**(告诉你"该往哪走")这两个独立通道为用户导航——这种把语义放在听觉、把方向放在触觉的分工,在保持认知负担低的同时,把耳朵留给真实世界的环境声(对盲人和低视力用户来说,环境声本身就是关键的感知通道)。

名字来自 *roam* 这个词根——"想去哪儿就去哪儿"的自由和尊严感。中文:**漫行**(màn xíng)。

## 这个项目【是】和【不是】什么

**是:**

- 一个**电子辅助出行工具(ETA, Electronic Travel Aid)**,**增强**白盲杖和导盲犬,而不是替代它们。
- 一个**多模态、多设备**系统。语音和触觉协同工作。形态从手机(V0)逐步演进到智能眼镜(V3)。
- **纯货架硬件**。不做电路板设计、不做固件烧录、不动烙铁。所有元件都是今天就能买到的消费品:一部 Android 手机、一个 Shokz 骨传导耳机、一只 Bangle.js 2 智能手表,以及将来的一副 Mentra Live 智能眼镜。
- **AI 协作开发**。项目的核心目标之一,是验证一个小团队(甚至一个人)+ Claude,能不能把过去需要整个工程组才能交付的无障碍软件,做出来、上线、并持续维护。

**不是:**

- 不是医疗器械。Roana 不诊断、不治疗、也不治愈任何东西。
- 不替代定向行走(O&M)训练、白盲杖、或导盲犬。
- 不是又一个把视觉大模型(VLM)塞进盒子里的产品。核心安全循环是一个轻量的"目标检测 + 单目深度"管线以约 10 FPS 运行;视觉大模型只在按需场景调用(比如用户主动问"前面是什么?")。

## 整体架构

```
┌──── 输入 ────┐   ┌──── 计算 ────┐   ┌──── 输出 ────┐
│ 手机摄像头    │ → │ Android NPU   │ → │ 骨传导 TTS    │
│ (V3: 眼镜)   │   │ YOLO11n-seg   │   │ (语义)        │
│ 手机 IMU      │   │ Depth Anything│   │ Bangle.js 2   │
│              │   │ 几何决策器     │   │ 震动 (方向)   │
└──────────────┘   └──────────────┘   └──────────────┘
```

三层:

1. **感知层** — YOLO11n-seg(目标检测 + 可行走区域分割,约 6 MB)+ Depth Anything V2-Small(约 2500 万参数,单目深度,骁龙 8 Gen 3 NPU 上约 50 ms)。常驻运行约 10 FPS。
2. **决策层** — 把深度图 + 可行走区域投影成 15×15 网格;DFS 搜索最长安全前进走廊;输出 `直行 / 左转 / 右转 / 停止` 之一,配合 IMU 朝向稳定化。
3. **反馈层** — 语音(骨传导 TTS)负责语义("前方有台阶,2 米");震动(左右腕 Bangle.js 2 触感)负责方向和紧急信号。

重型 VLM(SmolVLM2 / Gemini Nano)按需触发,不每帧调用。

## 设计原则

1. **只用货架硬件。** 每个元件必须今天就能在淘宝 / 京东 / 亚马逊 / 官方商店买到。不做硬件开发。
2. **全无线。** 没有线缆。对盲人用户来说这是硬性体验底线。
3. **最快验证优先。** 每个阶段都产出一个真实视障用户能拿去走的东西,1–8 周之内。
4. **语音和震动是并列的两个主通道**,不是"语音是 V0 临时凑数"。语音负责语义、震动负责方向,合在一起让认知负担最低,把耳朵留给世界。
5. **以手机为基础,服务眼镜形态。** V0–V2 的代码在 V3(摄像头从手机换到眼镜)时不需要改动。
6. **安全 > 覆盖度。** 当置信度下降(弱光、玻璃门、下行楼梯)时,系统优雅降级并明确告知用户该依赖盲杖了。
